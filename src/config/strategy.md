# Strategy

This is a low-frequency, aggressive-growth rollout strategy for a small dedicated Robinhood Agentic Account.
It is designed for scheduled checks every 30 minutes, not for fast stop-losses, event-chasing, or intraday scalping.
The goal is higher upside capture through carefully gated small positions, not uncontrolled trade frequency.

> Path note: file references below use the runtime-block variable names (e.g. `TODAY_ALLOWLIST_PATH`,
> `DAILY_PLAN_PATH`, `DSA_SIGNALS_PATH`, `DAILY_USAGE_PATH`) that the runtime block injects. They
> resolve at run time to the dated run folder `runtime/state/runs/<date>/planner|signals/...` — do not
> assume the old flat `runtime/state/<file>` paths.

Universe:
- Premarket source universe: `src/config/universe.txt`.
- Optional research signal layer: `DSA_SIGNALS_PATH`, generated from Daily Stock Analysis-inspired strategy lenses.
- Daily trade allowlist: `TODAY_ALLOWLIST_PATH`, generated by the premarket agent.
- Daily watchlist: `DAILY_PLAN_PATH (today_watchlist field)`.
- Intraday may trade only the intersection of those files.

Priority themes for the premarket screen:
- AI semiconductors and compute.
- AI data-center infrastructure, networking, power, and cooling.
- CPO, photonics, interconnect, and narrow-gate supply-chain candidates.
- Space, defense, drones, satellites, and autonomy.
- Nuclear, uranium, power, and energy infrastructure.
- Broad beta ETFs only when single-name conditions are weak or market risk is elevated.

General:
- Only act when `DAILY_PLAN_PATH` allows the action.
- Only act on symbols in `TODAY_ALLOWLIST_PATH` and today's `today_watchlist`.
- Only act in the dedicated Robinhood Agentic Account.
- Prefer high-conviction small orders over broad low-quality activity.
- If there are no strong setups, do nothing.

Daily dynamic allowlist construction:
- Select 3-8 symbols from `src/config/universe.txt`.
- Read `DSA_SIGNALS_PATH` when available, same-day, and valid.
- Use DSA signals as research evidence only; they are not trading permission.
- Rank sectors first, then symbols.
- Score each candidate 0-100:
  - theme momentum 20
  - catalyst/news quality 20
  - technical confirmation 20
  - liquidity/tradability 15
  - portfolio/account fit 15
  - downside/volatility control 10
- 80-100: tradable candidate if risk rules pass.
- 65-79: watch only unless price confirms strongly.
- Below 65: do not include in today's trade allowlist.
- Block symbols with unverified catalysts, earnings gap risk, liquidity concerns, unclear tradability, or stale quote data.

Daily Stock Analysis signal layer:
- The DSA layer is inspired by https://github.com/ZhuLinsen/daily_stock_analysis but runs through Codex subscription via `codex exec`.
- It does not run the third-party project's LLM API stack by default.
- It writes `DSA_SIGNALS_PATH`.
- It may use these lenses:
  - `hot_theme`: theme heat and whether the company is a real beneficiary.
  - `event_driven`: dated catalyst, credibility, impact path, and whether it is priced in.
  - `bull_trend`: trend alignment and relative strength.
  - `shrink_pullback`: controlled pullback inside an uptrend.
  - `volume_breakout`: breakout or reclaim with volume confirmation.
  - `growth_quality`: growth quality, cash flow, valuation, and durability.
  - `dragon_head`: sector leader behavior during active rotation.
- DSA signals can adjust candidate priority:
  - `promote`: candidate can receive extra confidence if Robinhood data and risk rules agree.
  - `neutral`: no special adjustment.
  - `demote`: candidate should usually remain watch-only.
  - `block`: candidate should be blocked unless fresh verified evidence invalidates the block.
- DSA signals must never override:
  - Robinhood account identification;
  - quote/tradability checks;
  - `KILL_SWITCH`;
  - trading mode;
  - risk tier caps;
  - allowlist intersection;
  - limit-order-only rule;
  - tax/wash-sale blocks when available.

Candidate buckets:
- `core_beta`: SPY, QQQ, SMH, VOO, IWM. Use when single-name quality is weak or market risk is elevated.
- `ai_infra`: AI semiconductors, data-center infrastructure, networking, photonics, power, and cooling.
- `high_beta_theme`: space, defense, autonomy, nuclear, uranium, and high-beta AI software.
- Prefer no more than 1-3 candidates from the same bucket in the final daily watchlist.

Buy rule:
- Only if `daily_plan.allowed_actions` includes `small_limit_buy`.
- Only if the symbol is in `TODAY_ALLOWLIST_PATH` and `today_watchlist`.
- For broad ETFs, prefer pullbacks: current price down at least 0.5%-1.0% from prior close or reclaiming intraday VWAP after weakness.
- For high-momentum single names, allow breakout/reclaim setups only if premarket plan marks `breakout_allowed=true` for that symbol and quote data is fresh.
- Do not buy if `market_regime` is `risk_off` or `no_trade`.
- Do not buy if the account already has an open order for that symbol.
- Do not average down automatically if the existing position is below cost basis.
- Limit price must be at or below the latest verified quote.
- Order notional must respect `src/config/risk.md`, `src/config/runtime.env`, and `DAILY_USAGE_PATH`.

Sell rule:
- Only sell existing long positions in the Agentic Account.
- Only if `daily_plan.allowed_actions` includes `partial_take_profit`.
- If a position is up more than 2.5% from cost basis, consider partial take profit.
- Risk exits are automatic in paper, not alert-only:
  - When technical invalidation levels exist and price breaks the `invalidation_below` level, the
    position is fully exited (`full_invalidation_exit`).
  - As a last-line safety net, any position whose loss vs average cost breaches the catastrophic
    hard stop (`HARD_STOP_LOSS_PCT`, default 8%) is fully exited even when no technical levels exist
    and even when the plan permits no discretionary sell action (`catastrophic_stop`). Set
    `HARD_STOP_LOSS_PCT=0` to disable.
  - These automatic exits run in paper only; review/live order placement remains human-gated
    (`execution_not_wired`).
- Never short.
- Limit price must be at or above the latest verified quote unless explicitly reducing risk after a review-safe take-profit setup.

Default:
- If unclear, do nothing.
- If data is missing, stale, conflicting, or hard to reconcile, do nothing.
- If no high-quality setup appears, append a `no_action` decision with the reason.
